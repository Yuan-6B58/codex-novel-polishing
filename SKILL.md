# 网络小说润色技能

## 工作流程

收到小说文本后，依次执行以下阶段。

### 阶段 0：作品分析

分析小说的题材类型与写作特点，以简短列表呈现，并主动询问用户确认或提供更精准方向：

**题材判断：**
- 都市、玄幻、仙侠、科幻、悬疑、言情、历史、游戏、轻小说……
- 流派：爽文、虐文、甜宠、逆袭、系统流、穿越、重生、无限流……

**写作分析：**
- 叙事视角：第一人称 / 第三人称 / 上帝视角
- 节奏倾向：快节奏爽文 / 慢热铺垫 / 张弛交替
- 语言风格：口语化 / 文艺 / 冷峻 / 热血

**输出格式：**
```
## 作品分析
- 题材：XXX
- 建议写作手法：XXX、XXX
- 当前节奏：XXX
- 语言风格：XXX

是否按此方向润色？或请指定偏好的方向。
```

### 阶段 1：逻辑 Bug 检查

逐项审查并指出问题，直接在文本中修复：

- 时间线矛盾：前文说三天后，后文变成一周后
- 人物关系冲突：同一人物的身份、能力前后不一致
- 设定自相矛盾：世界观规则被角色行为打破
- 情节因果断裂：事件A本应导致B，结果跳到了C
- 实力体系崩塌：战力忽高忽低

**输出**：列出发现的每个 Bug，给出修复建议，并在改后文本中直接修正。

### 阶段 2：去 AI 味

检测并修复以下 AI 写作痕迹：

**AI 味典型特征（需消除）：**
- "不仅……而且……""既……又……"等机械关联词堆积
- "首先……其次……最后……"流水账式推进
- 每段开头句式高度重复（主语+谓语+宾语）
- 缺乏语气变化，人物对白像说明书
- 情绪描写模板化："心中涌起一股暖流""眼神中闪过复杂之色"
- 过度使用高级词汇堆砌，词藻华丽但空洞

**修复手段：**
- 增加人物内心独白与心理活动
- 对白加入口语化处理：语气词、省略、打断
- 句式长短交错，避免连续三个句子同一结构
- 用具体的动作、细节代替抽象的形容词
- 加入作者独有的叙事口吻和节奏感

**目标**：让文本读起来像真人作者心流状态下逐字手打的产物。

### 阶段 3：增强吸引力

根据原作的类型，运用以下手法提升阅读粘性：

- **反差**：人物外表与内心的反差
- **对比**：强弱、贫富、善恶的极端对比制造戏剧张力
- **钩子/悬念**：每章末尾留一个未解问题或期待点
- **情绪过山车**：压抑-爆发-短暂喘息-更大危机
- **期待引导**：让读者知道前方会发生什么有趣的剧情
- **节奏控制**：紧张处短句快切，松弛处适当展开细节

不强行堆砌技巧，阅读流畅度优先。

### 阶段 4：字数达标与结构调整

要求每章至少 **2500 字**。

- 不足 2500 字：在情节薄弱处补充描写（环境、动作、心理），或展开次要情节
- 远超 2500 字但结构松散：适当精简冗余描写与游离情节
- 结构问题：检查章节断点是否在高潮/悬念处

不灌水凑字。

### 阶段 5：丰富剧情

在不大幅改变原有主线的前提下：

- 为人物行动补充合理动机
- 适当增加推动主线的支线细节
- 为配角增加鲜明的性格特征
- 重要情节转折前埋伏笔

**约束**：新增内容不超过原文字数的 15%，保持剧情紧凑。

### 输出格式

完成所有阶段后，输出：

```
## 作品分析
（题材、手法、节奏、风格分析）

## 修改说明
- 逻辑 Bug：[数量] 处，已修复
- AI 味处理：XX
- 吸引力增强：XX 手法
- 字数调整：原 XX 字 改为 修改后 XX 字
- 剧情补充：XX

## 润色后正文
（完整修改后的文章）
```
